# commits-names

branch sempre com o feature na frente

Conventional Commits:
feat - nova funcionalidade
fix - correção de bugs
docs - tudo que envolva somente documentação
style - alterações no estilo
test - criar ou aplicar testes
refactor - refazendo algo
perf - melhorias de perfomance e otimização
chore - qualquer coisa que não envolva código
